##### 晚上任务

```
flink on yarn 的四种部署方式
1.standalone
2.spark session 
前两种都是集群传输的时候浪费资源不适用
3.per-job 分配yarn,TaskManager,ResourceManager会公平分配（公平调度）调度yarn的时候需要网络传输因此使用与flink1.15.1之前的版本
4.Application 与 per-job 相似,但是数据会直接存储到HDFS中不需要网路传输而导致的数据丢失，因此公司更多的会选择这种方法
```

##### 内网穿透

```
内网穿透也叫做内网映射，也叫“NAT穿透”
一句话来说就是，让外网能访问你的内网；把自己的内网（主机）当成服务器，让外网能访问。
```

##### 使用内网穿透有什么好处？

```
没钱买服务器，自嗨用的
在开发的过程中，有时要用到外网能访问的地址进行一些功能测试，这时内网穿透就方便多了，节省时间
```
